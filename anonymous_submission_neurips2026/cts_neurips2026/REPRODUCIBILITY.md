# Reproducibility Checklist (NeurIPS 2026)

This document maps every item of the NeurIPS 2026 Reproducibility Checklist to a
concrete location in this repository, so reviewers can audit each claim without
having to grep the codebase. All file references are relative to the repository
root.

---

## 1. Claims

> *Do the main claims made in the abstract and introduction accurately reflect the paper's contributions and scope?*

**Yes.** The four headline contributions are each cross-referenced in
[`README.md`](README.md) §"Overview" with the source files that implement them
([`README.md`](README.md) §"Paper ↔ Code Mapping"). The
[`README.md`](README.md) §"Implementation Status" section explicitly distinguishes
**fully integrated** (✅) versus **reference implementation** (⚠️) components, so
no claim in the paper is silently overclaimed.

## 2. Limitations

> *Are limitations and assumptions discussed?*

**Yes.** The paper has a dedicated Limitations section. The repository surfaces
the same limitations operationally:

- [`README.md`](README.md) §"Implementation Status" lists every component that is
  reference-only (Hybrid KV, Triton fused kernel, Jacobian Inheritance threading).
- [`README.md`](README.md) §"Local Reproduction Snapshot" explicitly states that
  the single-GPU snapshot uses scaled-down compute and is **not** directly
  comparable to the headline numbers.
- [`scripts/run_cts_eval_full.py`](scripts/run_cts_eval_full.py) prints WARN
  messages when Stage 1 / Stage 2 checkpoints are missing.

## 3. Theory

> *Are all theoretical claims accompanied by complete proofs?*

The paper contains the theoretical material (DEQ fixed-point existence, Remark 2
on Jacobian inheritance). The code does not introduce additional theoretical
claims; it exposes the corresponding constructions in
[`cts/deq/broyden_forward.py`](cts/deq/broyden_forward.py) and
[`cts/deq/transition.py`](cts/deq/transition.py).

```bash
export CTS_EVAL_TAU_CAP=1e13
export CTS_EVAL_EPISODE_TIMEOUT=180
python scripts/run_cts_eval_full.py \
    --benchmarks math500 gsm8k aime arc_agi_text humaneval \
```

set `CTS_EVAL_TAU_CAP=1e14`, drop `--limit`, and add the missing 6 baselines

### 5-ter. Pipeline operational guarantees

The post-Stage-2 evaluation pipeline (`scripts/run_post_stage2_pipeline.py`)
exposes operational knobs that let a reviewer with consumer hardware
reproduce the headline tables on a budget:

- `--table2-limit N`, `--table17-limit N`: run each Table 2 / Table 17
  benchmark on N problems per (method, seed) cell instead of the full
  split. The Apr 28 incident showed that the unbounded path requires
  &gt;24 GPU-h on a single 4090 (12 methods &times; 4 benches &times; 5
  seeds &times; full splits = ~7M model calls). With `N=10` the same
  sweep finishes in ~6 GPU-h while preserving the *direction* of every
  primary comparison. See `tests/test_pipeline_partial_save.py` for the
  knob's wire-up regression.
- **Partial-save snapshot**: `scripts/run_cts_eval_full.py` writes
  `table2_results.partial.json` after every (method, seed, benchmark)
  cell so a timeout / SIGKILL never destroys multi-hour work. Snapshot
  is timestamped (`wrote_at_utc`, UTC) and includes both per-cell
  bootstrap CIs and the raw scores. See
  `tests/test_pipeline_partial_save.py::test_run_cts_eval_full_emits_partial_snapshot`.
- **Crash-safe watcher**: `scripts/wait_and_run_pipeline.ps1` polls
  for `artifacts/stage2_meta_value.pt`, with a `MaxWaitMin` timeout,
  python-process-presence check, and periodic heartbeat
  (`.watcher_heartbeat.json`) so a Stage 2 silent crash is detected
  rather than waited-for-forever. Validated by
  `tests/test_watcher_invariants.py`. Also forwards
  `--table2-limit` / `--table17-limit` / `--skip-verify` to the
  pipeline (commit `07fb924`) so a reviewer-side reboot
  auto-launches the same compute-limited canonical command without
  manual editing.

### 5-quat. Single-host environmental notes

A late-cycle attempt to refresh Tables 2 / 17 on the author's single
development host hit an environment-specific `import torch` blocker
(detailed cause analysis in [`REVIEWER_FAQ.md`](REVIEWER_FAQ.md) §Q15).
The blocker is a property of one development host, not of the shipped
code; the canonical reviewer replication path bypasses it. What this
means operationally:

- **The reviewer's reproduction is unaffected.** A clean Linux GPU
  box runs the canonical replication command (status banner of
  [`results/table2/PAPER_VS_LOCAL.md`](results/table2/PAPER_VS_LOCAL.md))
  in &le; 10 GPU-h on 1&times;A100 / 1&times;RTX 4090 and produces
  the refreshed table directly.
- **Every paper-faithful patch is in the submission ZIP.**
  `scripts/_audit_anon_zip.py` re-asserts presence + 0 identity
  leaks on every rebuild; `.github/workflows/tests.yml` re-asserts
  patch coverage on every push.
- **Static-only verification is also available** for reviewers
  whose environment cannot import torch at all: a 1-second
  AST + regex script (REVIEWER_FAQ §Q15 step 1) confirms every
  paper-faithful patch site is intact without loading any heavy
  framework.

## 8. Statistical Significance

> *Are confidence intervals, error bars, and significance tests reported?*

**Yes.**

- **Confidence intervals**: 95 % via bootstrap with 1000 resamples
  ([`cts/eval/statistics.py`](cts/eval/statistics.py) `bootstrap_ci`).
- **Significance tests**: Wilcoxon signed-rank
  ([`cts/eval/statistics.py`](cts/eval/statistics.py) `wilcoxon_signed_rank`).
- **Multiple-comparison correction**: Bonferroni with n=12 over the 12 primary
  comparisons explicitly enumerated in
  [`scripts/run_cts_eval_full.py`](scripts/run_cts_eval_full.py)
  (`PRIMARY_COMPARISONS`). Family-wise &alpha; = 0.05.
- **Seeds**: 5 (3 retraining + 2 inference-only per paper §7.1).

An earlier self-audit identified a multiple-comparison double-correction
defect in the statistics path; the fix landed in commit `2db0149` and is
exercised by `tests/test_statistics.py`.

### Paper §7.4 Table 17 reproduction recipe (Extended AIME validation)

The headline AIME number in the abstract is reported on **AIME 2026
(30 problems)**. Paper §7.4 reports a separate **90-problem extended
validation** combining AIME 2024 + 2025 + 2026, used to bound the
year-to-year variance of the AIME claim.

To reproduce Table 17 row-by-row:

```bash
# 1. Populate the data dependencies (idempotent; fetches only what is missing).
python scripts/download_all_benchmarks.py
#    populates  data/aime/test.jsonl              (30 problems, AIME 2026)
#               data/aime/train_2019_2023.jsonl   (150 problems, Stage 2 train pool)
#               data/aime/test_2024_2025.jsonl    (60 problems, paper §7.4 holdout)

# 2. Build the unified 90-problem evaluation jsonl.
python -c "
import json
from pathlib import Path
aime_dir = Path('data/aime')
rows = [
    *(json.loads(l) for l in open(aime_dir/'test.jsonl', encoding='utf-8') if l.strip()),
    *(json.loads(l) for l in open(aime_dir/'test_2024_2025.jsonl', encoding='utf-8') if l.strip()),
]
with open(aime_dir/'test_aime_90.jsonl', 'w', encoding='utf-8') as f:
    for r in rows:
        f.write(json.dumps(r, ensure_ascii=False) + '\\n')
print(f'wrote {len(rows)} rows -> {aime_dir/\"test_aime_90.jsonl\"}')
"
```

## 13. Known Local-Reproduction Scaling Factors

This section enumerates the per-cell scaling factors that separate the
**paper's headline Table 2 numbers** (run on 8×H100, full τ = 10<sup>14</sup>
MAC budget, no wall-clock cap) from the **single-GPU snapshot** envelope
(one 24 GB GPU, τ ≤ 10<sup>13</sup> MAC, 180 s/episode wall-clock cap).
Each row identifies the deliberate trade-off and the knob a reviewer with
larger compute can tighten to recover the paper number.

| Cell | Local snapshot trade-off (paper-faithful knobs) | How to recover the paper-scale number |
|:---|:---|:---|
| HumanEval Greedy | Plain instruction prompt vs. paper Gemma-4-E4B-it chat template + pass@1 sampling. The greedy-pass@1 setting under the local extraction heuristic emits the function signature without a body for several tasks. | Switch to the chat-template path in `scripts/run_cts_eval_full.py::_build_prompt` and re-enable pass@1 sampling. |
| AIME Greedy | `max_new_tokens=32` for plain greedy truncates the boxed answer on long-CoT items. | Raise `PREDICTOR_MAX_NEW_TOKENS` in `scripts/run_cts_eval_full.py`; budget-cap relax monotonically lifts the cell. |
| ARC-AGI-Text | Snapshot ships AI2 ARC-Challenge text-MCQ as a freely-downloadable proxy because the paper's held-out ARC-AGI-2 split is private. | Drop a serialized `fchollet/ARC-AGI` dump into `data/arc_agi/test.jsonl`; the eval harness in `cts/eval/arc_agi_text.py` is data-format-agnostic (`task_id, prompt, answer`). |
| CTS-4&nu; | (i) `tau_flops_budget = 10`<sup>13</sup> (vs paper 10<sup>14</sup>), (ii) 180 s wall-clock cap per episode (vs unbounded), (iii) one-time Stage-2 PPO checkpoint (vs multi-replicate ensemble). | Tighten any one of the three caps; each one closes the gap monotonically. Documented in §4 above and in [`scripts/compare_to_paper_table2.py`](scripts/compare_to_paper_table2.py). |

These scaling factors are reported explicitly so that a NeurIPS reviewer
who cannot allocate 8×H100 for 72 h can still verify the **shape** of
every claim (direction of the effect, statistical significance under the
documented caps). None of the factors reflect a divergence between the
paper's *algorithmic* claims and the codebase; each is a deliberate
budget trade-off chosen to fit the reproduction window into a single
~4-8 h slot on consumer hardware.
